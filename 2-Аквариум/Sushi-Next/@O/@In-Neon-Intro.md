# Neon — что это и зачем (очень простыми словами)

**Neon** — это облачный Postgres (база данных), сделанный так, чтобы дружить с современным деплоем (в т.ч. с Vercel).

Если кратко:

- **Postgres** — “настоящая” SQL-база данных
- **Neon** — сервис, который даёт Postgres как облако: создаёшь проект → получаешь строку подключения → вставляешь в `DATABASE_URL`

## Нужно ли Neon прямо сейчас?

Нет.

`body/` уже настроен так, что **без `DATABASE_URL` проект запускается** (DB будет просто “выключен”), а UI и `/api/health` будут работать.

Neon нужен, когда ты хочешь:

- хранить пользователей/данные
- подключить Drizzle по-настоящему
- потом делать миграции/таблицы

## Как подключить Neon (минимальный путь)

1) Зарегистрируйся в Neon (обычно удобно через GitHub)
2) Create Project
3) Внутри проекта найди **Connection string** (обычно начинается с `postgres://...` или `postgresql://...`)
4) На Vercel добавь Environment Variable:

- Key: `DATABASE_URL`
- Value: (строка подключения из Neon)

5) (Опционально) локально:

- заполни `body/.env` → `DATABASE_URL=...`
- попробуй `npm run db:push` (создаст таблицы по схеме)

## Важно

- Для Vercel переменные окружения задаются в UI: Project → Settings → Environment Variables
- Для продакшна и превью обычно ставят одинаковые значения (или разные, если хочешь разные базы)

## Что будет «достаточно» для первого деплоя

Можно вообще не подключать Neon и просто деплоить, чтобы увидеть сайт.

А уже потом, когда будет комфортно, подключим Neon и включим реальные таблицы.
