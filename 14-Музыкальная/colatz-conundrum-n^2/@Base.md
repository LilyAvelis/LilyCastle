Новая задача: формализовать и исследовать динамику центральной "области света" в квадратичном Коллатце.

Исходные условия (оставляем как в текущем проекте):

- Правило:
  - если n чётное: T(n) = n / 2
  - если n нечётное: T(n) = a _ n^2 + b _ n + c
- Параметры симуляции:
  - a ∈ [-2, 2], b ∈ [-10, 10], c ∈ [-20, 20] (или близкие диапазоны, которые уже использовались)
  - стартовые n: 1..100
  - max_steps ≈ 10 000
  - max_value ≈ 1e12
  - метрика: средний log(steps + 1) по стартам

Цель этой задачи: НЕ делать ещё один общий обзор,
а СФОКУСИРОВАТЬСЯ на центральной области света (область быстрой сходимости возле (a≈0, b≈0))
и формально описать её динамику по c и по threshold.

Нужно:

1. Модель данных

- Чётко зафиксируй, как мы определяем "область света":
  - возьми несколько уровней порога по метрике (например, quantile или фиксированные значения),
  - для каждого порога выдели связную компоненту вокруг (a≈0, b≈0),
  - работай именно с ЭТОЙ компонентой (центральный кластер), а не со всем шумом.
- Для каждого значения c и для каждого выбранного порога сохрани:
  - площадь этой области (в координатах (a, b)),
  - приблизительные границы (min/max a, min/max b),
  - может быть, простые параметры формы (например, соотношение сторон, эксцентриситет через аппроксимацию эллипсом).

2. Временной ряд по c

- Рассматривай c как "время" (ось эволюции).
- Для фиксированных порогов построй:
  - график площади области света как функции c,
  - графики min/max a(c) и min/max b(c) (как "дыхание" области по ширине/высоте).
- Найди:
  - диапазоны c, где область света существует,
  - точки, где она появляется/исчезает,
  - где она раздваивается или резко меняет форму (бифуркации).

3. Формализация динамики

- Попробуй описать динамику области света словами и числами, а не только картинками:
  - "При c в диапазоне [c1, c2] область света расширяется по a, оставаясь узкой по b."
  - "При c > c\*, область разваливается на несколько компонент" (если это происходит).
- Сделай отдельный markdown-отчёт (например, @Out.md), где будет:
  - чёткое определение "области света",
  - таблицы/графики: площадь, min/max a, min/max b в зависимости от c,
  - перечисление ключевых режимов: появление, рост, сжатие, исчезновение.

4. Визуализации ТОЛЬКО под эту задачу

- Сделай 2D- и 3D-графики, которые помогают видеть именно ЭВОЛЮЦИЮ центра:
  - 2D: для нескольких фиксированных c нарисуй только центральную область света, без всей внешней смерти.
  - 3D: plot (c по оси X, a или b по оси Y, площадь или границы по оси Z), чтобы было видно, как "столб света" дышит при изменении c.
- Не распыляйся на новые общие красивые блинчики. Здесь задача — сделать ясную картину динамики ОДНОГО объекта.

5. Жёстко: никаких новых правил, никаких сглаженных косинусных Коллатцев.

- Используем ровно то же дискретное правило с if (even)/else (odd).
- Никаких f(x) с cos(pi \* x) и прочих “гладких костылей”.
- Задача — понять структуру реальной области света, а не чинить систему.

Вывод, который мне нужен на выходе:

- Один .md-файл с формализованным описанием области света и её динамики:
  - определение,
  - числовые характеристики (площадь, границы) как функции c,
  - словесное описание фаз: где свет рождается, где растёт, где умирает.
- Набор минимально необходимых графиков, иллюстрирующих эти фазы.
- Краткий комментарий исследователя в @Comments: как он это видит, что его удивило и какие дальше естественные вопросы к этой области.
