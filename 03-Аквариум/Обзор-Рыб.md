# Обзор Рыб в Аквариуме

## Список Рыб

- **trout (Форель)**: Next.js — React-based фреймворк с SSR/SSG. Хорош для больших apps, гибкий.
- **mackerel (Скумбрия)**: Nuxt — Vue-based фреймворк с SSR/SSG. Удобный, с отличными dev tools (Vue DevTools в браузере).
- **perch (Окунь)**: SolidJS — реактивный фреймворк с JSX. Быстрый, как Svelte, но с React-подобным синтаксисом.
- **smelt (Корюшка)**: Astro — Мульти-фреймворковый для статических сайтов с "островами" компонентов. Идеален для SEO и простых страниц.
- **shark (Акула)**: Angular — Enterprise фреймворк с TypeScript, dependency injection, RxJS. Хорош для крупных приложений с строгой архитектурой.

## Поведение с Симлинками (node_modules из Библиотеки)

- **Astro (smelt)**: Живёт нормально — работает с симлинком после добавления `fs.allow` в `astro.config.mjs`.
- **Next.js (trout)**: Вредная рыба — не работает с симлинком (Turbopack блокирует внешние пути). Требует локальные `node_modules` в каждом проекте.
- **Nuxt (mackerel)**: Живёт нормально — работает с симлинком без дополнительных настроек.
- **SolidJS (perch)**: Живёт нормально — работает с симлинком.
- **Angular (shark)**: Вредная рыба — симлинк требует прав администратора (SymbolicLink). Использует локальные `node_modules`.

## Рекомендации

- Для оптимизации места: Симлинки для Astro, Nuxt, SolidJS. Локальные для Next/Svelte/Angular.
- Библиотека (`01-Библиотека/js-libs`) — центральный источник пакетов для обновлений.
- Low tier: Astro (smelt) — для "поиграться".
- Mid tier: Nuxt (mackerel) — для проектов с dev tools.
- High tier: Next.js (trout) — для enterprise, Angular (shark) — для строгих архитектур.

## Вывод

Все проекты оставлены как наследие успешного тестирования фреймворков в Аквариуме. Для будущих ИИ:

- Большие проекты: Nuxt (mackerel) — гибкий, с отличными dev tools.
- Мелкие проекты: Astro (smelt) — быстрый, SEO-friendly для статических сайтов.
- Средние проекты: SolidJS (perch) — реактивный, быстрый, с JSX-синтаксисом.
