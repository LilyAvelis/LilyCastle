# Отчет по Bug Manager Next UI

## Что сделано
- Создан полноценный Next.js интерфейс `BugManager-Next` (TypeScript + Tailwind) с тёмной тактической темой.
- API `/api/bugs` переписан на Node.js: теперь он обходит все комнаты формата `NN-Комната`, читает `.bug` файлы и возвращает список комнат с полными метаданными жуков.
- Клиентская страница `src/app/page.tsx` отображает 4×4 (адаптивную) сетку карточек комнат: название, количество жуков, список с эмодзи и языками, пустые комнаты выделяются отдельным состоянием.
- Кнопка "Shuffle Жуков" вызывает `/api/shuffle`, показывает прогресс и перезапрашивает актуальные данные без мерцания.
- Добавлены понятные статусы загрузки, ошибки и результаты перемешивания.
- Проект успешно проходит `npm run lint`.

## Как запускать / использовать
1. Открой PowerShell и перейди в каталог проекта:  
   `cd "c:\Users\Public\LilyCastle\Тронный-зал\BugManager-Next"`
2. Установи зависимости (однократно):  
   `npm install`
3. Запусти дев-сервер:  
   `npm run dev`
4. Открой адрес, который Next.js выведет в терминале (обычно `http://localhost:3000`, при занятости порта он подберёт `3001+`).
5. В интерфейсе видна живая карта комнат. Кнопка **Shuffle Жуков** вызывает Python-скрипт из `ЖучиныйМенеджер/bug-shuffle.py` и обновляет карту.

## Текущие ограничения
- API пока только читает данные: нельзя редактировать отдельного жука или массово менять `.bug` свойства.
- Перемещение жуков реализовано через существующий Random Shuffle; целенаправленного переноса между комнатами нет.
- Нет веб-форм/диалогов для редактирования описаний, языков и т.д.
